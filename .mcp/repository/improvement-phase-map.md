# Improvement Phase Map

This map gives ChatGPT and Codex a phase-by-phase path for growing ChornPlanet without breaking current pages.

## Phase 1: Repository Understanding And Maps

- Maintain `.mcp/repository/` as the practical codebase map.
- Keep `.codex/Agents.md` as Codex execution rules.
- Update maps when route, locale, metadata, content, or server architecture changes.

## Phase 2: SEO/LLM Visibility

- Review metadata coverage and sitemap accuracy.
- Draft public-safe `llm.txt` and `llm-full.txt`.
- Improve canonical topic hubs before adding high-volume content.

## Phase 3: Content Draft Foundation

- Define draft content entities and review workflows.
- Keep generation separate from publication.
- Store source tracking for factual media content.

## Phase 4: Outfit/Civilization Draft Pipeline

- Follow DNA authority.
- Enforce `Zone List -> Zone File -> exactly one Sub-Zone`.
- Generate drafts only until external posting is approved.

## Phase 5: Metadata To MongoDB Migration

- Identify duplicated hardcoded metadata.
- Design typed metadata service/repository contracts.
- Migrate one page family at a time with all 10 locales.

## Phase 6: Analytics Growth Foundation

- Preserve existing Vercel Speed Insights and Google Analytics behavior.
- Add read-only reporting first.
- Avoid production tracking changes without approval.

## Phase 7: TikTok-First Commerce Linking

- Start with approved product/category landing pages and draft links.
- Avoid automatic product publishing or customer/order mutation.
- Keep commerce calls to action clear and localized.

## Phase 8: Daily Media Digest Drafts

- Build source-reviewed draft workflows.
- Separate search, summary, image prompt, video prompt, and publication approval.
- Avoid copying source articles.

## Phase 9: Smart Food UX Planning

- Plan login, favorites, ordering, booking, and preference flows.
- Treat customer identity, order, and payment data as high risk.
- Keep Smart Food production mutation behind explicit approval.

## Phase 10: Controlled Automation

- Add automation only after policies, audit logs, approvals, and rollback paths exist.
- Default to draft/review modes.
- Keep external posting and commerce mutation disabled unless explicitly approved.
